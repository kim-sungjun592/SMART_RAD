# 프로젝트 업데이트 내역

## 2026-07-21 업데이트 요약

### 1. 로그인 페이지 (`/login`)
- **불필요한 필드 제거**: 기존에 존재하던 '회사 도메인' 입력 항목을 제거하여 로그인 절차를 간소화했습니다.
- **자동완성 해제**: 브라우저의 폼 자동완성 기능으로 인해 아이디와 비밀번호가 잘못 채워지는 현상을 방지하고자 `autoComplete="off"` 속성을 폼 전체와 각 입력 필드에 적용했습니다.
- **로고 링크 적용**: 화면 좌측 상단의 TSM 로고를 클릭하면 랜딩 페이지(메인 화면)로 바로 이동하도록 수정했습니다.

### 2. 회원가입 페이지 신규 구현 (`/signup`)
- **레이아웃 통일**: 로그인 페이지와 동일한 분할 레이아웃(Split-pane layout)을 적용하여 일관된 디자인을 유지했습니다.
- **입력 필드 구성**: 이름, 이메일, 비밀번호, 비밀번호 확인, 학교명(기존 '회사명'에서 변경) 필드를 구현했습니다.
- **텍스트 변경**: 기존 '무료로 시작하기' 텍스트를 용도에 맞게 '회원가입 신청하기'로 변경했습니다.
- **자동완성 해제**: 로그인 페이지와 마찬가지로 브라우저 자동완성을 차단했습니다.

### 3. 랜딩 페이지 (`/`)
- **Scroll to Top**: 상단 헤더의 TSM 로고 클릭 시 화면이 부드럽게 맨 위로 스크롤되도록(smooth scroll) 기능을 개선했습니다.

### 4. 인사기록 관리 페이지 (`/employees`)
- **가입 승인 UI 추가**: '교직원 정보관리' 페이지 상단의 '+ 교직원 등록' 버튼 좌측에 **'교직원 가입 승인'** 버튼을 새롭게 배치했습니다.
- **승인 대기자 팝업(Modal)**: 가입 승인 버튼 클릭 시 대기 중인 사용자의 정보(이름, 이메일, 학교명, 신청일)를 한눈에 볼 수 있는 팝업 모달(`SignupApprovalModal`)을 구현했습니다.
- **임시(Mock) 연동**: 실제 백엔드 연동 전 프론트엔드에서 동작을 확인할 수 있도록 가상의 대기자 데이터와 승인/거절 처리용 Mock API 로직을 구성해 두었습니다.
