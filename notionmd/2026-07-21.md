### 1. 회원가입 페이지 신규 구현 (`/signup`)
- 로그인 페이지와 동일한 **분할 레이아웃(split-pane)** 적용으로 디자인 일관성 확보
- 입력 필드: **이름 / 이메일 / 비밀번호 / 비밀번호 확인 / 학교명** (기존 '회사명' → '학교명'으로 교직원 도메인에 맞게 변경)
- CTA 문구를 '무료로 시작하기' → **'회원가입 신청하기'** 로 변경 (승인제 가입 성격 반영)
- 브라우저 자동완성 차단(`autoComplete="off"`)

### 2. 교직원 가입 승인 (인사기록 관리 `/employees`)
- '교직원 정보관리' 상단 '+ 교직원 등록' 버튼 좌측에 **'교직원 가입 승인'** 버튼 추가 (대기 건 표시용 빨간 점 배지 포함)
- 클릭 시 **승인 대기자 모달**(`SignupApprovalModal`) 노출 — 이름 / 이메일 / 학교명 / 신청일 목록과 **승인·거절** 액션 제공
- 승인·거절 시 확인창 후 목록에서 즉시 반영
- **임시(Mock) 연동**: 대기자 조회·승인·거절을 프론트 Mock 함수로 구성(`getPendingSignups` / `approveSignup` / `rejectSignup`) — 실제 API 연동 전 동작 확인용

### 3. 로그인 페이지 개선 (`/login`)
- **'회사 도메인' 입력 필드 제거** → 로그인 절차 간소화 (아이디/비밀번호만)
- 브라우저 자동완성으로 값이 잘못 채워지는 문제 방지: 폼 `autoComplete="off"`, 비밀번호 `autoComplete="new-password"`
- 좌측 상단 **TSM 로고 클릭 시 랜딩 페이지(`/`)로 이동**
- 다크모드에서 텍스트가 안 보이던 문제 대응(명시적 글자색 지정)

### 4. 랜딩 페이지 (`/`)
- 상단 헤더 **TSM 로고 클릭 시 부드럽게 최상단으로 스크롤**(smooth scroll) 하도록 개선

### 5. 공통 UI 다크모드 가독성 (`components/ui.tsx`)
- `Input` / `Select` / `Field` 공통 컴포넌트에 글자색·배경색 명시 → 다크모드에서 흰 배경 위 흰 글씨로 안 보이던 현상 해결
