# 작업 로그 관리 가이드 (WORKLOG GUIDE)

이 저장소의 작업 기록은 두 폴더로 나눠 관리한다. **다음 에이전트/팀원은 이 문서를 먼저 읽고 규칙에 맞게 작성한다.**

## 폴더 구조
- **`notionmd/`** — *날짜별* 노션 업로드용 요약. 파일명 `YYYY-MM-DD.md`.
  - 그 날 **팀 전체**가 한 일을 날짜 단위로 정리(담당자는 실명으로 표기).
- **`workMd/`** — *개발자별* 상세 작업 로그. 파일명 `YYYY-MM-DD_<GitHub핸들>.md`.
  - 예: `2026-07-21_hyunminboo.md`(부현민), `2026-07-22_MJB9595.md`(김재아).
  - 상단에 작성자·날짜 메타 헤더를 둔다.

## 새 작업 md 만드는 절차 (에이전트용)
1. 대상 날짜와 작성자(GitHub 핸들)를 정한다.
2. 해당 작성자의 그 날 커밋을 조회한다:
   ```
   git log --all --no-merges --author="<핸들>" \
     --since=<YYYY-MM-DD> --until=<다음날> \
     --date=format:'%Y-%m-%d %H:%M' --pretty='%ad | %s'
   ```
3. 커밋들을 사람이 읽기 쉽게 요약(무엇을 / 왜 / 결과).
4. `workMd/YYYY-MM-DD_<핸들>.md` 로 저장(상단 메타 헤더 포함).
5. 필요 시 그 날 전체를 `notionmd/YYYY-MM-DD.md` 에 담당자 실명과 함께 종합.

## 규칙
- 커밋이 **없는 날은 기록하지 않는다** (예: 2026-07-13 커밋 없음).
- 커밋 날짜(작성일) 기준으로 배치한다. 실제 작업일과 커밋일이 다르면 커밋일을 따르되 비고에 적는다.
- 담당자 표기는 반드시 위 매핑표의 실명을 사용한다.
