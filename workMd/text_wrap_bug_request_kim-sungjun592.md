> 요청자: 김재아 (MJB9595) · 2026-07-22
> 분류: UI/UX 버그 (프론트 전반) · 담당: 로그인/공통 UI를 맡은 김성준

## 증상
전체 프론트엔드 여러 화면에서 좁은 컬럼/셀 안의 텍스트가 **글자 단위로 줄바꿈**됨.
- 예(교직원 가입 승인 모달): `김재아` → `김재/아`, `경복대학교` → `경복대/학교`, `2026-07-22` → `2026-/07-22`
- 테이블 셀, 카드 값 등에서 창 폭이 줄면 특히 심함.

## 원인 (조사 결과)
- 전역 `word-break: break-all` / `break-words` 같은 규칙은 **없음**(globals.css 확인).
- 실제 원인은 **컬럼/컨테이너 폭 부족 + 처리 미흡**:
  - CJK(한글)는 기본적으로 글자 사이 어디서나 줄바꿈 가능.
  - 날짜/숫자는 하이픈(`-`)에서 줄바꿈됨.
  - 좁은 grid/table 컬럼에 `white-space` 처리가 없어 그대로 접힘.

## 권장 수정 (전역)
1. 짧은 고정값 셀(이름·사번·날짜·상태·구분 등)에 `whitespace-nowrap` 적용.
2. 또는 셀 `min-width` 지정 / 테이블 레이아웃 조정으로 최소 폭 확보.
3. 모달·카드 컨테이너 폭을 콘텐츠에 맞게 충분히 확보(지나치게 좁은 grid 지양).
4. 정말 긴 텍스트(메모/설명)만 `break-words` 허용하는 식으로 정책 분리.

## 범위 / 경계
- **전역(여러 페이지 테이블·카드)** 은 김성준님 담당 UI라 이 요청으로 남깁니다.
- **교직원 가입 승인 관련 프론트**(현재 김재아 작업 영역)는 **이미 로컬 수정 완료**:
  - `SignupApprovalModal` 폭 확대(`w-[880px] max-w-[95vw]`) + 각 셀 `whitespace-nowrap` 적용.
  - 참고 커밋/파일: `frontend/src/components/SignupApprovalModal.tsx`.
